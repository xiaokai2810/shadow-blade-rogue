# 迭代记录

## V0.2/V0.3 合并迭代：菜单、角色、排行榜、怪物生态

### 产品 Agent

目标：把游戏从单一开局扩展为有出战选择、成长差异、排行榜目标和怪物识别度的可持续体验。

本轮范围：

- 主菜单增加多页签：出战、角色、排行、图鉴。
- 增加本地排行榜，记录分数、角色、时间、击杀、等级。
- 增加三名可选角色，并让角色有不同初始属性和专属玩法。
- 增加多种怪物强度和行为，形成普通、迅捷、精英、远程、重甲、护卫、Boss 的生态。
- 强化动态视觉表现：角色光环、怪物飘动、护盾、冲锋预警、莲华阵、连锁电弧、Boss 震荡环。

不做项：

- 暂不接入联网排行榜。
- 暂不新增付费、广告或账号系统。

### 架构 Agent

核心方案：

- 使用 `CHARACTERS` 配置表承载角色属性、描述、专属天赋和颜色。
- 使用 `ENEMY_TYPES` + `ENEMY_ORDER` 配置表承载怪物属性、权重、行为和图鉴文本。
- 新增 `LEADERBOARD_KEY` 和 `SELECTED_CHARACTER_KEY` 本地存储。
- 继续保持网页、抖音小游戏、Android 三端共享 `blade-survivor.js`。

风险控制：

- 新增素材使用独立 `*-v2.png` 文件，保留旧 sprites 作为回退资源。
- 新增敌方子弹数组 `enemyProjectiles`，与玩家子弹隔离。
- 排行榜只存本地，不传输用户数据。

### 美术 Agent

本轮视觉策略：

- 使用 imagegen 生成 4x3 角色/怪物精灵图集，并用本地 chroma-key 去背流程切成独立 PNG。
- 二次按主体连通区域重切精灵，清理固定网格裁切造成的断边和邻格残片。
- 保留已有主角、怪物、Boss、飞刃图片作为回退，降低资源风险。
- 用 Canvas 动态效果做差异化：角色光环、怪物护盾、冲锋角标、毒灯亮点、夜魇相位环、莲花瓣、雷电弧。
- 主菜单、角色页、图鉴页和战斗场景都接入新精灵，让角色选择和怪物强度更容易识别。
