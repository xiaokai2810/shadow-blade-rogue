# 游戏多 Agent 迭代池

本文件把多 Agent 协作体系应用到 `影刃花海` 的后续开发。每个任务都先由主 Agent 负责定义，再交给下游 Agent 处理。

## V0.2: 局内成长深度

目标：让每局 5 分钟内出现更明显的流派选择，而不是只做数值叠加。

| ID | 主 Agent | 任务 | 交付物 | 验收 |
| --- | --- | --- | --- | --- |
| V0.2-P1 | 产品 Agent | 定义 3 个武器流派：飞剑、旋刃、落雷 | 需求卡 | 每个流派有核心爽点、升级路线和克制场景 |
| V0.2-A1 | 架构 Agent | 设计技能/升级数据结构 | 技术方案 | 新增技能不需要改大量条件分支 |
| V0.2-D1 | 开发 Agent | 实现流派成长和升级权重 | 代码 | 玩家 3 局内能明显打出不同构筑 |
| V0.2-T1 | 测试 Agent | 回归升级、Boss、死亡重开 | 测试报告 | 无卡死、无无限升级、无伤害 NaN |
| V0.2-R1 | Review Agent | 检查性能和多端同步 | Review 结论 | 无 P0/P1 阻断 |

## V0.3: 怪物和 Boss 内容扩展

目标：让敌人不只是血量差异，而有移动、攻击、压迫节奏差异。

| ID | 主 Agent | 任务 | 交付物 | 验收 |
| --- | --- | --- | --- | --- |
| V0.3-P1 | 产品 Agent | 定义怪物阵营和 Boss 行为 | 需求卡 | 至少 4 种普通怪、2 个 Boss 技能 |
| V0.3-Art1 | 美术 Agent | 设计怪物视觉层级 | 素材清单 | 小怪、精英、Boss 在 1 秒内可识别 |
| V0.3-A1 | 架构 Agent | 设计敌人行为配置 | 技术方案 | 新增敌人不破坏旧敌人生成逻辑 |
| V0.3-D1 | 开发 Agent | 实现冲锋、远程、护甲、召唤行为 | 代码 | 每类敌人有独立战斗感 |
| V0.3-T1 | 测试 Agent | 验证敌人生成和难度曲线 | 测试报告 | 前 3 分钟不突然秒杀，Boss 可被击杀 |

## V0.5: 多端发布自动化

目标：减少网页、抖音小游戏、Android 三端不同步。

| ID | 主 Agent | 任务 | 交付物 | 验收 |
| --- | --- | --- | --- | --- |
| V0.5-A1 | 架构 Agent | 设计共享源码同步策略 | 技术方案 | 一条命令同步三端核心逻辑 |
| V0.5-D1 | 开发 Agent | 增加同步/构建脚本 | 脚本 | `npm run check` 能发现三端源码不一致 |
| V0.5-T1 | 测试 Agent | 建立发布前检查清单 | 测试清单 | GitHub Pages 和服务器发布前可自动验证 |
| V0.5-R1 | Review Agent | 审查密钥、产物、发布配置 | Review 结论 | 仓库无密钥、无 build 垃圾产物 |

## 当前优先级建议

1. 先做 `V0.2`，因为它直接提升游戏可玩性。
2. 再做 `V0.5`，因为三端同步越晚自动化，后续改动成本越高。
3. `V0.3` 和 `V0.4` 可以并行准备产品和美术方案，但开发不要同时铺开。
